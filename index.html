Core Concepts

Two Ideas That Change Everything

Before writing code, understand the two breakthroughs that make GenLayer unique.

Concept 01

⚖️ Optimistic Democracy

When your contract calls an AI model, a randomly-selected group of validators each run the contract independently — each with their own AI (GPT, LLaMA, Mistral…).

One validator is the Leader: they run first and propose a result. Other validators verify it. If the majority agree, the result is accepted on-chain.

💡 Analogy Like a court: one judge writes the verdict, a panel independently reviews. If the majority agree, it becomes law — fast, fair, trustless.
```
Concept 02

🔀 Equivalence Principle

Five validators might get five differently-worded LLM responses. "Yes", "Definitely!", "Affirmative" — all mean the same thing but aren't identical strings.

The Equivalence Principle defines what counts as agreement. You choose the rule in your contract.

💡 Analogy Like grading exams: two students say the same thing differently. The rule defines whether they both pass.
```
Mode When to Use Example
strict_eq Byte-for-byte identical results required Boolean True/False, exact numbers
non_comparative Each validator checks their own result against a rule "Does the response contain 'YES'?"
comparative Second LLM call checks if two results mean the same thing Long-form text, subjective answers

Key Concepts in the Contract

gl.exec_prompt()

Sends your prompt to the validator's configured LLM. Each validator uses their own AI model — GenLayer routes it automatically.

gl.eq_principle_strict_eq()

Wraps your AI function. Runs it in the non-deterministic sandbox and requires all validators to return the exact same boolean.

gl.DynArray[type]

A dynamic array that GenLayer serializes and stores on-chain. Survives between calls forever, unlike a regular Python list.

@gl.public.write vs .view

.write changes state and requires a transaction. .view is read-only and free — no gas, no signing.

Connect with genlayer-js

This is how your website talks to the blockchain. Three core functions you'll use in every GenLayer dApp.

📦 Install npm install genlayer-js — or fork the official boilerplate which has it pre-configured.
📤 writeContract() — Send a Question
// Sends a write transaction
const txHash = await client.writeContract({
  address:      CONTRACT_ADDRESS,
  functionName: 'ask_question',
  args:         [question],

// Wait for AI validators to agree
await client.waitForTransactionReceipt({

oracle.js — Full Setup 
import { createClient, createAccount } from 'genlayer-js';
import { simulator } from 'genlayer-js/chains';
import { TransactionStatus } from 'genlayer-js/types';

// ── Config ──────────────────────────────────────────────
const CONTRACT_ADDRESS = ‘0xYOUR_CONTRACT_ADDRESS_HERE’;

// ── Create client connected to Studio simulator ──────────
const account = createAccount();
const client  = createClient({ chain: simulator, account });

// ── Ask a question ───────────────────────────────────────
async function askQuestion(question) {
await client.initializeConsensusSmartContract();
const txHash = await client.writeContract({
address: CONTRACT_ADDRESS, functionName: ‘ask_question’,
args: [question], value: 0n,
});
return client.waitForTransactionReceipt({
hash: txHash, status: TransactionStatus.FINALIZED,
retries: 100, interval: 3000,
});
}

// ── Read the latest answer ───────────────────────────────
async function getLatest() {
return client.readContract({
address: CONTRACT_ADDRESS, functionName: ‘get_latest’,
args: [], stateStatus: ‘accepted’,
});
}

// ── Read all answers ────────────────────────────────────
async function getAllAnswers() {
return client.readContract({
address: CONTRACT_ADDRESS, functionName: ‘get_all_answers’,
args: [], stateStatus: ‘accepted’,
});
}
